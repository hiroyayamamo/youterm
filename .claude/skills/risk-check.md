スキル名: /risk-check
説明: PJの現状コンテキストに対してリスクチェックを実行し、見落としているリスクを洗い出す
前提: CLAUDE.mdにルーティングルールとKey Filesが定義されていること

## いつ使うか

- フェーズ変化時（要件定義→設計、設計→開発、リリース前、受入期間開始等）
- 「なんか嫌な予感がする」時
- 定例前のセルフチェック
- クライアントの態度や体制に変化があった時

## 手順

### 1. PJ構造の把握

まずCLAUDE.mdを読み、以下を特定する：
- **ルーティングルール** — 情報がどこに格納されているか（ログ、データ、レポートの格納先）
- **Key Files** — 参照すべきファイルの一覧
- **現状サマリー** — PJの現在の状況
- **鉄則** — PJ固有のルール

### 2. 現状コンテキストの収集

CLAUDE.mdのルーティングルールに従い、以下に該当するファイルを探して読む：

| 情報の種類 | CLAUDE.mdのルーティングルールで格納先を確認 |
|-----------|----------------------------------------|
| MTG決定事項・合意事項 | 意思決定ログ |
| スコープ・機能合意の履歴 | スコープログ |
| 契約状態・未解決の宿題 | 契約ログ |
| ステークホルダーの最新状態 | ステークホルダーログ |
| 予算の消化状況 | 予算ログ |
| 直近のやり取り（Slack等） | 生データ格納先（最新50行程度） |
| 直近のレポート | レポート格納先 |

※ファイルが存在しない項目はスキップする。全てのPJに全てのログがあるとは限らない。

### 3. ナレッジの参照

CLAUDE.mdのKey Filesから、以下に該当するナレッジファイルを探して読む：

| ナレッジの種類 | 典型的なファイル名 |
|--------------|-----------------|
| チェックリスト | ai-pm-requirements-checklist等 |
| PM運営の方法論 | pm_playbook等 |
| ドキュメント運用ガイドライン | pm-document-guideline等 |
| リスク評価・商務 | pm_risk_and_commercial等 |

※Key Filesに上記に該当するファイルがない場合は、refs/ディレクトリを探索する。

### 4. リスクチェック実行

Step 2で把握した現状に対して、Step 3のナレッジを照合し、以下を出力する：

#### A. 今のフェーズで確認すべきなのに未確認の項目
チェックリストの中から、現在のフェーズに該当する項目を抽出し、PJの記録と照合する。記録がない＝未確認として報告する。

#### B. 直近のやり取りから読み取れるリスク兆候
PM方法論のパターン（書面回避、体制変更、ドキュメント攻撃等）に該当する動きがないかチェックする。

#### C. ドキュメント運用の遵守状況
ドキュメントガイドラインの必須セットに対して、現在のフェーズで出すべきドキュメントが作成・共有されているか確認する。

#### D. 契約・商務上のリスク
契約の曖昧さ、未払いリスク、スコープクリープの兆候がないかチェックする。

### 5. 出力フォーマット

```markdown
## リスクチェック結果（{日付}）

### 現在のフェーズ: {要件定義/設計/開発/リリース/受入}

### 🔴 即時対応が必要
- {具体的なリスクと根拠。根拠はログのファイル名・日付を明記}

### 🟡 注意・監視が必要
- {具体的なリスクと根拠}

### 🟢 問題なし
- {確認済みの項目}

### 推奨アクション
1. {最優先でやるべきこと}
2. {次にやるべきこと}
```

### 6. 結果の格納

CLAUDE.mdのルーティングルールに従い、レポート格納先に `risk-check-{YYYYMMDD}.md` として保存する。
重要なリスクが見つかった場合は、該当するログファイルにも追記する。

## 注意点

- チェックリストの全項目を毎回チェックする必要はない。現在のフェーズに関連する項目に絞る
- 「問題なし」の項目も記録する。後から「確認していなかった」と言われないため
- 推測と事実を明確に分ける。ログに記録がある事実と、やり取りから推測されるリスクは区別する
- このスキルはリスクの検出まで。対処方針の策定はユーザーが判断する
- PJにナレッジファイルが存在しない場合は、一般的なPM知識に基づいてチェックする。ただしその旨を出力に明記する
