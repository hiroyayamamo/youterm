スキル名: /add-log
説明: logs/配下のファイルにフォーマット済みエントリを追記する
前提: 追記すべき情報（事実・決定・変更等）が会話中に存在すること

手順:
1. 追記対象のログファイルを特定する（CLAUDE.mdのルーティングルールに従う）
   - 意思決定・合意 → logs/meeting_decisions.md
   - スコープ変更 → logs/scope_log.md
   - 予算・見積り → logs/budget_log.md
   - 技術選定 → logs/tech_decisions_log.md
   - ステークホルダー → logs/stakeholders.md
   - 契約 → logs/contract_log.md
2. 対象ファイルを開き、既存テーブルのフォーマットを確認する
3. 以下の必須項目を含むエントリを作成する:
   - 日付: YYYY-MM-DD
   - ソース: 情報の出所（MTG名、メール、Slack、資料名など）
   - 信頼度: 確定合意 / 口頭合意 / こちらの認識 / 先方主張 / 推測
   - 内容: 事実と判断を分離して記述
4. テーブルの末尾に追記する（既存エントリは上書きしない）
5. 追記内容を簡潔に報告する

## 信頼度の判断基準
- 確定合意: 書面（契約書、メール）で合意済み
- 口頭合意: MTG等で口頭合意。議事録に記載
- こちらの認識: 自分の解釈。先方の明示的合意なし
- 先方主張: 先方が主張しているが、こちらは合意していない
- 推測: 状況からの推測。要確認

## なぜこのスキルが必要か
ログエントリのフォーマット漏れ（日付なし、信頼度なし等）が起きると、後から情報の確度を判断できなくなる。フォーマットをスキルに組み込むことで、毎回WRITING_RULESを参照せずとも品質を担保する。
