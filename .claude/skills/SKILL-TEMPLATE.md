# スキルテンプレート

> 新しいスキルを作るときはこのテンプレートをコピーして使う。
> ファイル名は `{skill-name}.md`（例: `transcribe-and-update.md`）

```
スキル名: /skill-name
説明:   このスキルが何をするかの1行要約
前提:   実行前に満たすべき条件（例: 対象日のページがNotionに作成済みであること）
手順:
  1. 〇〇を minutes/ から取得
  2. □□を reports/ から取得
  3. 取得した情報を統合・変換
  4. 出力先（Notion/Slack/Gmail等）に書き込む
```

## 設計ルール
- 各手順で入力元（どのフォルダ/サービス）と出力先を明示する
- 1スキル1機能（UNIX哲学）。複合処理は複数スキルの組み合わせで実現する
- スキル作成後、なぜこのスキルが必要か・どの業務課題を解決するかをこのファイルに追記する
