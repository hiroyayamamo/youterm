スキル名: /extract-decisions
説明: 議事録（minutes/のテキストまたは会話中のMTGメモ）から決定事項を抽出し、ルーティングルールに従って各ログファイルに振り分ける
前提: 議事録テキストが会話中に存在する、またはminutes/に保存済みであること

手順:
1. 議事録テキストを取得する
   - 会話中に貼り付けられた場合: そのまま使用
   - ファイルの場合: minutes/から読み込む
2. テキストから以下のカテゴリの情報を抽出する:
   - **意思決定・合意**: 「〜で進める」「〜に決定」「〜で合意」等の表現
   - **スコープ変更**: 「〜を追加」「〜は除外」「〜はPhase2へ」等
   - **予算関連**: 金額、見積もり、追加費用への言及
   - **技術選定**: ツール名、フレームワーク名、アーキテクチャへの言及
   - **ステークホルダー変化**: 人事異動、担当変更、新メンバー追加
   - **契約関連**: 締結、変更、検収への言及
3. 抽出されなかったカテゴリは明示的に「該当なし」と報告する
4. 各抽出項目に信頼度を付与する:
   - 「〜で合意」「〜に決定」→ 口頭合意
   - 「〜を検討する」「〜も考える」→ こちらの認識 or 推測
   - 「先方が〜と言っていた」→ 先方主張
5. /add-log スキルを使って各ログファイルに追記する
6. 抽出結果のサマリを報告する:
   - 追記先ファイルと件数
   - 信頼度が「推測」以下の項目（要確認リスト）

## 判断の分かれ目
- 議論があったが結論が出ていない項目 → ログに追記しない。「次回確認事項」として報告のみ
- 同一の決定が複数ログに該当する場合 → 両方に追記し相互参照（「→ scope_log」等）を付ける
- 発言者が不明確な場合 → 「MTG参加者」として記録し、信頼度を「こちらの認識」にする

## なぜこのスキルが必要か
MTG後にログを更新する作業は毎週発生するが、手動だと抽出漏れ・振り分け間違いが起きる。特に「議論はしたが合意していない」ものを合意として記録する誤りが危険。スキルで振り分けルールを統一することで品質を担保する。
