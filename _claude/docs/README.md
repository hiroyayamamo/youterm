# docs/ — 業務棚卸し

定期的にClaude Codeに「今の業務を整理して」と頼んで、まとめさせた結果が蓄積される。
自分が何をやっているかの棚卸し用。新メンバーのオンボーディング資料にもなる。

## 置くもの
- 業務フロー全体の可視化結果
- 週次サイクルの俯瞰図
- 情報フロー図（どの情報がどのスキルを通ってどう変換されるか）
- オンボーディング用ドキュメント

## 作り方
スキル群・議事録・レポートが蓄積されたタイミングで、AIに全ファイルを読ませて
「自分の業務を分析して」と頼む。業務フロー全体がリバースエンジニアリングされる。

## Index
| ファイル | 概要 |
|---|---|
| CLAUDE-EXAMPLE.md | CLAUDE.md記入例（架空PJ「SmartOrder」）。新規PJ立ち上げ時の参考 |

## 生成の目安
以下が揃ったタイミングで「業務を分析して」とAIに依頼する:
- minutes/ に議事録が5件以上蓄積
- logs/ に各ファイル10エントリ以上
- reports/ に週次レポートが3件以上
- .claude/skills/ にPJ固有スキルが追加されている

## 運用ルール
- 最初から作ろうとしない。スキルとminutes/reports/が十分溜まってから生成する
- 定期的に再生成して最新の業務状況を反映する
