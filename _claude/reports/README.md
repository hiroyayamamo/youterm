# reports/ — 加工済みアウトプット

minutes/の一次情報を分析・統合して、ステークホルダーに届けるための形にしたもの。

## 置くもの
- プロジェクト状況報告書
- 週間ワークフロー定義
- ステークホルダー向け資料

## スナップショット型 vs 更新型の使い分け

このフォルダには性質の異なる2種類のファイルが混在する。運用を間違えると情報が散逸する。

### スナップショット型（日付付き新規ファイル、上書きしない）
時点の記録が重要なもの。過去のバージョンに戻って参照する可能性がある。
- 週次報告
- 状況更新
- ステークホルダー向け資料

### 更新型（1ファイルを更新し続ける、常に最新が正）
合意・決定の現在地が重要なもの。「今どうなっているか」が1箇所でわかる必要がある。
- 合意事項・決定事項トラッカー
- スコープ定義（現在の合意済みスコープ）
- 役割分担・体制表

## 命名規則
```
# スナップショット型
{YYYY-MM-DD}_{レポート種別}.md
例: 2026-03-13_weekly-report.md

# 更新型
{トピック名}.md（日付なし）
例: agreements.md, scope.md, team-structure.md
```

## Index
| ファイル | 種別 | 概要 |
|---|---|---|
| weekly-report-TEMPLATE.md | テンプレート | 週次レポートの雛形。/weekly-report スキルで自動生成可能 |

## 運用ルール
- minutes/の情報を元に作成する（reports/が情報の出発点にならない）
- スナップショット型は日付付きで新規作成。上書きしない
- 更新型は同一ファイルを更新。変更履歴はgitで追跡する
- 外部向けに変換する際のフィルタリングルールはCLAUDE.mdの「報告フォーマット」を参照
