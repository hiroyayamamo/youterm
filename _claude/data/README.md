# data/ — 生データ・エビデンス原本

プロジェクトの生データとエビデンス原本を格納する。3つの役割がある。

## 置くもの

### 1. 構造化データ（CSV等）
- CSV、TSV等の分析用データ
- 集計用のSQLクエリ（.sql）

### 2. エビデンス原本（先方が改変・削除できるソースの保全）
- 先方管理の文書（見積書・発注書・覚書・要件書PDF等）
- 先方のGoogle Sheets/Notionのスナップショット（PDF化して保管。共有リンクは相手が変更可能）
- AI生成ドキュメントの受領版（各バージョン）
- 社内作成の一次資料（MTG準備、内部メモ、提案ドラフト等。加工前の状態で保全）

### 3. Slackログ
- `slack_raw.md` — Slackの原文をそのまま蓄積する追記型ファイル。ログへの転記とは別に原文を保全する

## 運用ルール
- 生データは加工せずそのまま保管する（加工結果はreports/に置く）
- データの出自（どこから取得したか、いつ時点のものか）をファイル名または先頭コメントに記録する
- 機密データ・個人情報を含むファイルは置かない（.gitignoreに追加）
- 相手が改変・削除できるソース（共有ドキュメント等）は受領時に即保存する
