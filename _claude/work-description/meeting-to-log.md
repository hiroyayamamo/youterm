# タスク定義書: 議事録からのログ抽出

## 概要
MTG終了後、議事録（minutes/）から意思決定・合意事項・スコープ変更等を抽出し、適切なログファイルに振り分ける。

## トリガー
- 議事録がminutes/に保存された時
- MTGの文字起こしテキストを受領した時

## 入力
- 議事録テキスト（minutes/に保存済み）

## 手順
1. 議事録を読み、以下を抽出する:
   - 意思決定・合意事項 → logs/meeting_decisions.md
   - スコープ変更 → logs/scope_log.md
   - 予算に関する合意・変更 → logs/budget_log.md
   - 技術選定・設計判断 → logs/tech_decisions_log.md
   - ステークホルダー情報 → logs/stakeholders.md
   - 契約関連 → logs/contract_log.md
2. 各エントリに日付/ソース/信頼度を付与する
3. /add-log スキルでフォーマットを担保しつつ追記する

## 判断ポイント
- **信頼度の判定**: MTGでの口頭合意は「口頭合意」。「検討する」「考える」は「こちらの認識」または「推測」
- **複数ログにまたがる情報**: 同一の決定が複数ログに該当する場合（例: 技術変更がスコープに影響）、両方に追記し相互参照リンクを付ける
- **不明確な発言**: 合意に至っていない議論は追記しない。次回MTGの確認事項として記録

## 出力
- 各ログファイルへの追記（/add-log スキル経由）
- 抽出結果の簡潔な報告

## 対応スキル
- /add-log（エントリ追記）
- /extract-decisions（議事録からの抽出自動化）
