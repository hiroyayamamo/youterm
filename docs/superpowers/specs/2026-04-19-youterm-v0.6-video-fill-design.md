# youterm v0.6 設計書 — Video-Fill Mode

**作成日**: 2026-04-19
**対象**: youterm v0.6.0
**前提**: v0.5.1 がリリース済み

## 1. 概要

YouTube の動画を youterm ウィンドウ全域に最大化表示する「Video-Fill モード」を追加。OS レベルのフルスクリーンではなく、ウィンドウ内部でのみ最大化(youterm ウィンドウをリサイズすれば動画サイズも追従)。ショートカット `Cmd+Shift+F` でトグル、状態は永続化。

ダーク/シアターモードの不在を補う代替体験として位置付ける。

## 2. スコープ

### 含む

- `Settings.videoFillMode: boolean` 追加、初期値 `false`
- `Cmd+Shift+F` でトグル
- ON 時: YouTube iframe 内に CSS 注入 → ヘッダ / サイドバー / コメント等を `display:none`、`#movie_player` を `position:fixed; inset:0; width:100vw; height:100vh`
- ON 時: Terminal overlay(タブバー含む)も非表示
- iframe ナビゲーション後も維持(YouTube 動画切替時の DOM 再生成対応)
- 起動時に前回の videoFillMode 状態を復元

### 含まない

- 動画プレイヤー制御(再生/一時停止/音量)の別 UI — YouTube の標準プレイヤーコントロールをそのまま使用
- 動画以外のページ(home / search)で ON 時の専用挙動 — 動画 player がない場合は単純に黒画面になる(ユーザーが watch ページへ navigate すれば機能する)
- 他モード(`youtube-only` / `overlay` / `terminal-only`)の廃止 — video-fill は**直交する独立トグル**

## 3. アーキテクチャ

### 3.1 直交モデル

video-fill は既存 `AppState.mode` とは独立した `Settings` 上の boolean。

- ON: mode に関係なく全視覚的要素(terminal overlay / タブバー / mode indicator / options panel)が非表示、YouTube iframe は chrome 無しで video のみ表示
- OFF: 以前の mode に戻る(既存 `AppState.mode` そのまま)

### 3.2 CSS 注入先と制御

YouTube iframe 内に永続的な `<style>` タグを 1 つ差し込み、`html.youterm-video-fill` クラスの有無で表示制御を切替。

**注入タイミング**:
- `attachYoutube` 内で `did-frame-finish-load`(YouTube iframe 側)を購読
- 毎ロードで `<style>` タグの存在を確認し、無ければ差し込む(idempotent)
- `videoFillMode` の state に応じて `html.youterm-video-fill` クラスを add/remove

**CSS 内容(最小構成)**:

### 3.3 Renderer 側(Terminal overlay)

`body.video-fill` クラスを追加し、`#terminal-root` と `#tab-bar` を `display:none`。

CSS:
```css
body.video-fill #terminal-root {
  display: none !important;
}
```

`#terminal-root` にはタブバー・mode indicator・options panel も含まれるので一括制御可。

## 4. IPC

### 新規

- `video-fill:toggle` (renderer → main) — ショートカットを menu 以外から起動する場合の余地(panel のトグル UI 追加時など)。v0.6 では renderer から呼ばないが API として用意

### 既存活用

- `settings:changed` ブロードキャスト:設定変更時、renderer は `body.video-fill` クラスを settings.videoFillMode に応じて toggle
- Main も同 subscribe で iframe の `html.youterm-video-fill` クラスをトグル

## 5. ショートカット

| Accelerator | 動作 |
|---|---|
| `Cmd+Shift+F` | `videoFillMode` をトグル(`settings.dispatch({ type: 'set-video-fill', value: !current })`) |

Cmd+F は xterm / YouTube の find との衝突を避けるため使わない。

### `validateAndNormalize`

`videoFillMode` の field-by-field validation:
- 型が boolean 以外 / 欠落 → `INITIAL_SETTINGS.videoFillMode` (= false)

## 7. iframe ナビゲーション時の再注入

YouTube は動画切替時に `<html>` ごと再描画することはほぼないが、`<style>` タグが残っているか保険的に再チェック:

```ts
terminalView.webContents.on('did-frame-finish-load', async (_e, isMainFrame) => {
  if (isMainFrame) return
  const frame = findYoutubeFrame()
  if (!frame) return
  await ensureVideoFillStyleInjected(frame)
  await applyVideoFillClass(frame, settings.getSettings().videoFillMode)
})
```

### スタイル注入関数

```ts
async function ensureVideoFillStyleInjected(frame: Electron.WebFrameMain): Promise<void> {
  await frame.executeJavaScript(`
    (() => {
      if (document.getElementById('youterm-video-fill-style')) return
      const style = document.createElement('style')
      style.id = 'youterm-video-fill-style'
      style.textContent = \`${VIDEO_FILL_CSS.replace(/`/g, '\\`')}\`
      document.head.appendChild(style)
    })()
  `)
}

async function applyVideoFillClass(frame: Electron.WebFrameMain, enabled: boolean): Promise<void> {
  await frame.executeJavaScript(`
    document.documentElement.classList.toggle('youterm-video-fill', ${enabled})
  `)
}
```

## 8. テスト戦略

### ユニット(TDD)

- `settings.ts`:
  - `set-video-fill` action(true / false / same value で no-op / reset でデフォルトへ)
  - 予想追加 3-4 テスト
- `settingsStore.ts`:
  - `videoFillMode` の validation(boolean / 非 boolean / 欠落)
  - 予想追加 3 テスト

### 手動 / E2E

- `Cmd+Shift+F` で video-fill ON → 動画が窓いっぱいに表示される(ヘッダ/サイドバー/タブバー非表示)
- 再度 `Cmd+Shift+F` で OFF → 元のモードに戻る
- ON 状態で別動画へナビゲート → 再注入で維持される
- 再起動後も video-fill 状態が復元される

## 9. エラーハンドリング

- `webFrameMain.executeJavaScript` 失敗 → ログのみ、ユーザー体験は他の fallback で継続(CSS が入ってない分 UI は普通通り表示)
- YouTube の DOM セレクタ不一致(将来的な YouTube 改変) → 動画非表示/残存 UI あり、の可能性。機能劣化のみ、クラッシュはしない
