# youterm v0.5 設計書 — YouTube URL 永続化 + ログイン修正

**作成日**: 2026-04-18
**対象**: youterm v0.5.0
**前提**: v0.4.1 がリリース済み

## 1. 概要

2 つの改善を v0.5.0 でまとめる:

1. **最後に開いていた YouTube URL を起動間で復元**
2. **YouTube ログインボタン押下時のエラー修正**(accounts.google.com のフレームバスタ JS 対策)

## 2. スコープ

### 含む

- `Settings.youtubeLastUrl: string | null` 追加(初期 null = homepage)
- iframe のナビゲーションを main 側で監視し、YouTube 系 URL 変更時に settings を更新(debounced)
- 起動時、persisted URL があれば iframe の初期 src に使う(無ければ `https://www.youtube.com/`)
- ログイン動線: `accounts.google.com/*` へのフレームナビゲーションを検知 → `event.preventDefault()` → 新規 BrowserWindow を開いてログイン
- ログインポップアップ: 同じ session(`persist:terminal`)を使用 → Cookie 共有 → ログイン成功で YouTube 本体に反映
- ポップアップは `youtube.com/*` に戻ったら自動クローズ、Terminal 側の iframe を reload して認証状態を反映

### 含まない(将来)

- ログイン状態 / ダークモード / シアターモードの明示的な UI 表示(すでに session cookie で auto-persist)
- 複数 Google アカウント切替の UI
- 動画個別ページ URL の履歴(最後の1件のみ保持)

### URL validation

`validateAndNormalize` に youtubeLastUrl を追加:
- null または有効な https URL with youtube.com ドメインのみ accept
- 無効値 / 欠落 → null にフォールバック(homepage 起動)

## 4. URL キャプチャ

### 監視ポイント

Terminal WebContentsView の `webContents.on('did-frame-navigate', ...)` イベントを利用:

```ts
terminalView.webContents.on('did-frame-navigate', (_e, url, httpResponseCode, httpStatusText, isMainFrame) => {
  if (isMainFrame) return                          // main frame は我々の terminal.html
  if (!isYoutubeNavigationUrl(url)) return         // google auth / static resources 除外
  settings.dispatch({ type: 'set-youtube-url', url })
})
```

`isYoutubeNavigationUrl`:
- `youtube.com` / `www.youtube.com` / `m.youtube.com` の host のみ対象
- サブリソース(ytimg.com 等)は除外
- 先頭が `https://` であること

`did-frame-navigate` は client-side SPA ナビゲーションも捕捉する。debounce は settings 側の 200ms でカバー。

### 適用

`settingsController` は既存の debounced-save パターンをそのまま使う。特別な処理は不要。

## 5. 起動時の URL 復元

renderer main.ts が iframe を作る前に `settings:get-initial` で Settings を取得 → `iframe.src = settings.youtubeLastUrl ?? 'https://www.youtube.com/'`。

ただし現在 `iframe` は HTML 静的記述(`<iframe src="https://www.youtube.com/">`)になっている。これを **動的生成**に切り替えて、initial settings で src を決める形にする必要あり。

## 6. ログインポップアップ

### 問題

`accounts.google.com` のログインページは以下のフレームバスタ JS を含む:

```js
if (window.top !== window.self) {
  window.top.location = 'https://www.google.com/'
}
```

CSP `frame-ancestors` と `X-Frame-Options: DENY` は header stripper で剥がしているが、JS 側の検知は剥がせない → iframe 内でページが壊れる / 遷移する。

### 対処

Terminal WebContentsView 側で iframe ナビゲーションを監視:

```ts
terminalView.webContents.on('will-frame-navigate', event => {
  if (isGoogleAuthUrl(event.url)) {
    event.preventDefault()
    openLoginWindow(event.url)
  }
})
```

`isGoogleAuthUrl`:
- `accounts.google.com` / `myaccount.google.com` を含む Google 認証系ホスト

### `openLoginWindow(url)`

```ts
function openLoginWindow(url: string): void {
  const loginWin = new BrowserWindow({
    width: 480,
    height: 640,
    parent: win,
    modal: false,                 // モーダルじゃないほうが自然
    autoHideMenuBar: true,
    webPreferences: {
      session: terminalSession,   // cookie を共有
      contextIsolation: true,
      nodeIntegration: false,
      sandbox: true,
    },
  })
  loginWin.loadURL(url)

  loginWin.webContents.on('did-navigate', (_e, currentUrl) => {
    if (isYoutubeHomeUrl(currentUrl)) {
      loginWin.close()
      // Terminal 側の iframe を reload して認証反映
      terminalView.webContents.send('youtube:reload')
    }
  })
}
```

`isYoutubeHomeUrl`: host が youtube.com 系で、パスが `/` or `/?...`。

### ポップアップが閉じた後

`terminalView.webContents.send('youtube:reload')` で既存のリロード IPC を再利用。renderer の `onYoutubeReload` ハンドラが iframe.src を再セットして認証済みのページを再描画する。

## 7. IPC

### 新規チャネル

なし。URL キャプチャは main 内で settings.dispatch を直接呼ぶ形。ログイン系は既存 `youtube:reload` を使い回す。

## 8. 後方互換

- `Settings.youtubeLastUrl` 欠落の v0.4.1 settings.json → `validateAndNormalize` が field-by-field で補完(null)。他フィールドは保持 ✓

## 9. テスト戦略

### ユニット(vitest、TDD)

- `src/main/settings.ts`:
  - `set-youtube-url` アクションの全パターン(null 設定、文字列設定、同値で no-op)
  - reset で null に戻る
- `src/main/settingsStore.ts`:
  - `youtubeLastUrl` の validation(null / 有効 URL / 不正値 / 欠落)

### 手動 / E2E

- スモーク E2E は維持(URL 追跡の検証までは自動化しない)
- 手動: 動画を開く → アプリ終了 → 再起動 → 同じ動画が再生できる
- 手動: Sign in ボタン → ポップアップ表示 → ログイン → ポップアップ自動クローズ → iframe がログイン状態で表示
