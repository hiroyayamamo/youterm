# youterm v0.7 設計書 — Ad Block

**作成日**: 2026-04-19
**対象**: youterm v0.7.0
**前提**: v0.6.4 がリリース済み

## 1. 概要

YouTube の広告を `@ghostery/adblocker-electron` でブロック。`persist:terminal` session に EasyList / uBlock フィルタを適用し、ネットワーク層で広告リクエストを遮断する。オプションパネルのチェックボックスで ON/OFF 切替、状態は永続化。

## 2. スコープ

### 含む

- `@ghostery/adblocker-electron` 依存追加
- `Settings.adBlockEnabled: boolean` 追加、デフォルト `true`、永続化
- 起動時: 設定に応じて session に blocker を enable/disable
- ランタイム切替: 設定変更時に blocker 状態を更新 + iframe リロード(反映のため)
- オプションパネルに Ad Block チェックボックス(Blur スライダーの下あたり)
- フィルタリストは起動時にダウンロード、以後キャッシュ(`app.getPath('userData')` 配下)

### 含まない(v0.7 範囲外)

- ショートカット(パネル UI のみ)
- SponsorBlock(スポンサーセグメント自動スキップ)
- カスタムフィルタルール管理 UI

## 3. 制約・限界(CHANGELOG にも明記)

YouTube は広告ブロック対策を強化しており、本機能は**広告の約 80〜95% をカット**する想定:
- **SSAI(Server-Side Ad Insertion)**: 広告が動画ストリームに直接挿入された場合、network 層での分離不能 → ブロック不可
- **adblock 検知**: YouTube が検知して動画再生を停止する警告画面を出す可能性あり
- **フィルタのアップデート遅延**: YouTube 側の広告配信ロジック変更から EasyList の追従に数日〜数週間かかる場合あり

完璧ではないが、体感上大半の広告はスキップされる。

## 5. 実装アーキテクチャ

### 新規: `src/main/adBlock.ts`

```ts
import { ElectronBlocker } from '@ghostery/adblocker-electron'
import type { Session } from 'electron'

export interface AdBlockController {
  setEnabled(enabled: boolean): Promise<void>
  dispose(): Promise<void>
}

export async function createAdBlockController(session: Session): Promise<AdBlockController> {
  const blocker = await ElectronBlocker.fromPrebuiltAdsAndTracking(fetch)
  let active = false

  return {
    async setEnabled(enabled: boolean) {
      if (enabled === active) return
      if (enabled) {
        blocker.enableBlockingInSession(session)
      } else {
        blocker.disableBlockingInSession(session)
      }
      active = enabled
    },
    async dispose() {
      if (active) blocker.disableBlockingInSession(session)
      active = false
    },
  }
}
```

### `attachYoutube` 拡張(または新 `attachAdBlock`)

シンプルに済ませるため `attachYoutube` に統合:

```ts
const adBlock = await createAdBlockController(terminalView.webContents.session)
await adBlock.setEnabled(settings.getSettings().adBlockEnabled)

const adBlockUnsub = settings.subscribe(async s => {
  await adBlock.setEnabled(s.adBlockEnabled)
  // setting 変更時はリロードして反映
  if (!terminalView.webContents.isDestroyed()) {
    terminalView.webContents.send('youtube:reload')
  }
})
```

`dispose` で `adBlockUnsub()` + `adBlock.dispose()` を呼ぶ。

### 注意点

- `ElectronBlocker.fromPrebuiltAdsAndTracking(fetch)` は初回ロード時に network 経由でフィルタをダウンロード(〜数 MB)。オフライン時はフォールバックが効かない → エラーをログだけ吐いて無効化で継続
- ブラウザの fetch が Electron main process 32+ で使えるはず。使えなければ `cross-fetch` など追加

### イベント

`optionsPanel.ts` に新規 callback `onAdBlockChange(enabled: boolean)` を追加、panel 内で checkbox の `change` イベントを購読。

`updateSettings(s)` 内で `checkbox.checked = s.adBlockEnabled` を同期。

### renderer main.ts 拡張

panel callback に:
```ts
onAdBlockChange: enabled => window.youtermAPI.settingsSetAdBlock(enabled),
```

### IPC

新規 channel: `settings:set-ad-block` (renderer → main、boolean payload)。

Preload API に `settingsSetAdBlock(enabled: boolean)` 追加。

### 手動

- Ad Block ON(デフォルト) + YouTube watch page → 広告がスキップされる(動画の開始が早い、サイドバー広告が消える)
- パネルで OFF に切替 → iframe リロード → 広告が通常通り表示される
- 再起動で ON/OFF 状態が復元
- 初回起動: フィルタリストダウンロード成功を main コンソールで確認
