# youterm v0.16 — 画面 2 分割(split panes)設計

作成日: 2026-04-21

## 概要

ターミナル領域を縦に 2 分割し、左右それぞれに独立したタブ群を持たせる。分割は `Cmd+D` でトグル、タブは左右のペイン間をドラッグ&ドロップで移動できる。分割状態・各ペインのタブ構成・分割比率は永続化され、再起動時に復元される。

## ゴール

- `Cmd+D` で縦分割トグル(分割 → 統合、統合 → 分割)
- 左右間で tab の DnD 移動
- 1 → 2 分割時に右 pane に新タブを自動生成
- 分割状態と各 pane の tabs を永続化
- splitter ドラッグで分割比率を変更可、比率も永続化

## 非ゴール

- 3 pane 以上の分割(将来的な拡張余地は残すが、v0.16 では 2 pane 固定)
- 水平分割(横分割)
- pane 自体の開閉に対するアニメーション
- ドラッグ中のゴースト(標準の browser drag image で十分)

## 1. データモデル

`src/shared/types.ts`:

```ts
export interface Pane {
  tabs: Tab[]
  activeId: string
}

export interface TabsState {
  panes: Pane[]           // length 1 = 分割なし / 2 = 分割中
  activePaneIndex: number // 0 or 1
  splitRatio: number      // 0〜1、左 pane の比率。default 0.5
}

export const INITIAL_TABS_STATE: TabsState = {
  panes: [{ tabs: [{ id: '1', customName: null, cwd: null }], activeId: '1' }],
  activePaneIndex: 0,
  splitRatio: 0.5,
}
```

既存の `Tab` 型(id, customName, cwd)はそのまま流用。

### migration

`tabsStore.load()` が旧 shape(`{ tabs, activeId }`)を検知したら新 shape に変換:

```ts
if ('tabs' in parsed && !('panes' in parsed)) {
  return {
    panes: [{ tabs: parsed.tabs, activeId: parsed.activeId }],
    activePaneIndex: 0,
    splitRatio: 0.5,
  }
}
```

新 shape のバリデーション:
- `panes` が配列、かつ length が 1 または 2
- `activePaneIndex` が有効な index
- `splitRatio` が 0.1〜0.9 の範囲(外れ値は 0.5 に clamp)

壊れた JSON / バリデーション失敗 → `INITIAL_TABS_STATE` にフォールバック。

## 2. Reducer actions

`src/main/tabs.ts`:

```ts
export type TabsAction =
  // 既存(pane-aware に書き換え)
  | { type: 'new-tab'; id: string }
  | { type: 'close-tab'; id: string }
  | { type: 'activate-tab'; id: string }
  | { type: 'rename-tab'; id: string; name: string | null }
  | { type: 'set-tab-cwds'; cwds: Record<string, string> }
  | { type: 'move-tab'; id: string; beforeId: string | null }
  // 新規
  | { type: 'split-panes'; newTabId: string }
  | { type: 'unsplit-panes' }
  | { type: 'activate-pane'; index: 0 | 1 }
  | { type: 'move-tab-to-pane'; id: string; paneIndex: 0 | 1; beforeId: string | null }
  | { type: 'set-split-ratio'; ratio: number }
```

### 振る舞いルール

- **`new-tab`**: `activePaneIndex` の pane に追加、その tab を activeId に。
- **`close-tab`**: 該当 tab が属する pane から削除。その pane が空になったら **auto-unsplit**(残った pane が唯一の pane に)。両 pane 合わせて 0 個になる場合は state を変えない(window close は controller 層の責務)。
- **`activate-tab`**: tab が属する pane を検出し、`activePaneIndex` をその pane に、pane の `activeId` を対象 tab に更新。
- **`move-tab`**: 同 pane 内での順序変更。既存の動きそのまま。
- **`split-panes`**: 既に分割中なら no-op。分割なし時は右 pane を作成し、`newTabId` を唯一の tab として持たせる。`activePaneIndex = 1` に。
- **`unsplit-panes`**: 分割なしなら no-op。右 pane の tabs を左 pane 末尾に concat。`activeId` は元 `activePaneIndex` の pane のものを維持。`activePaneIndex = 0`。
- **`activate-pane`**: index が現在と同じ、または存在しない index → no-op。
- **`move-tab-to-pane`**: 他 pane へ移動。同 pane 指定なら `move-tab` 同等に振る舞う。元 pane が空になったら **auto-unsplit**。移動後、対象 tab を移動先 pane の `activeId` に + `activePaneIndex` を移動先に。
- **`set-split-ratio`**: 分割なしなら no-op。ratio は 0.1〜0.9 に clamp。

### auto-unsplit の共通ヘルパー

`close-tab` と `move-tab-to-pane` で同じロジックを使うため、reducer 内に `collapseIfEmpty(state)` 相当のヘルパーを用意。

## 3. UI & レンダリング

### DOM 構造

分割なし時(panes.length === 1):

```html
<div id="terminal-root">
  <div class="pane" data-pane-index="0">
    <div class="tab-bar">...</div>
    <div class="terminal-inner">...xterm 群...</div>
  </div>
</div>
```

分割中(panes.length === 2):

```html
<div id="terminal-root">
  <div class="pane" data-pane-index="0">
    <div class="tab-bar">...</div>
    <div class="terminal-inner">...xterm 群...</div>
  </div>
  <div id="splitter"></div>
  <div class="pane" data-pane-index="1">
    <div class="tab-bar">...</div>
    <div class="terminal-inner">...xterm 群...</div>
  </div>
</div>
```

### xterm runtime の扱い

- `runtimes: Map<string, TabRuntime>` は変更なし(tab id → xterm instance、1 対 1)
- tab が pane 間移動 → `runtime.container` を移動先 pane の `terminal-inner` に `appendChild` で付け替え(xterm インスタンス自体は破棄しない)
- container 付け替え後、新しいサイズで `fit.fit()` を再実行 + `ptyResize` 送信

### フォーカス pane の視覚表現

- active pane の `.tab-bar` 下端: `rgb(40, 254, 20)` の明るい 1px
- inactive pane の `.tab-bar` 下端: `rgba(255, 255, 255, 0.15)` の抑えた 1px

### splitter

- 6px 幅の縦帯、cursor: col-resize
- ドラッグで `splitRatio` 更新(ドラッグ中は local state、release 時に IPC 送信)
- ratio clamp は 2 段構え:
  - **永続化 clamp**(reducer 側): 0.1 〜 0.9(外れ値は 0.5 に reset)
  - **runtime clamp**(renderer 側): 最小 pane 幅 200px で制限(ウインドウ幅が広い時は 0.1-0.9 と同等、狭い時はさらに制限されうる)
- CSS: `flex: 0 0 6px` で固定幅

### mode 連携

既存の body class (`mode-youtube-only` / `mode-overlay` / `mode-terminal-only`)に変更なし。分割 UI は `#terminal-root` 配下なので、mode-youtube-only 時には display:none で隠れる(既存動作のまま)。

## 4. IPC & API

`src/shared/types.ts` の `YoutermAPI` に追加:

```ts
panesToggleSplit(): void
panesActivate(index: 0 | 1): void
panesSetRatio(ratio: number): void
tabsMoveAcross(tabId: string, paneIndex: 0 | 1, beforeTabId: string | null): void
```

IPC channels:
- `panes:toggle-split` → main が id 採番 → `split-panes` / `unsplit-panes` dispatch
- `panes:activate`
- `panes:set-ratio`
- `tabs:move-across`

既存 `tabs:move`(同 pane 内)は残す。クロスペインは別 API にして意図を明確化。

### menu / shortcut

`src/main/shortcuts.ts` に追加:
- `Cmd+D` → `tabsController.toggleSplit()`

既存のショートカット(Cmd+1/2/3 モード、Cmd+Shift+F、Cmd+R、Cmd+T、Cmd+W 等)は変更なし。

### state broadcast

`tabs:state` のペイロードは新 shape になる。renderer は受け取り側で panes[].tabs を展開して描画。

## 5. 永続化

`tabsStore` の load / save は既存ロジックそのまま(`JSON.stringify(state)` 経由)。state shape が変わるだけ。

保存される項目:
- 分割状態(`panes.length`)
- 各 pane の tabs(順序・名前・cwd)
- 各 pane の activeId
- `activePaneIndex`(再起動時のフォーカス pane)
- `splitRatio`

debounced save(200ms)はそのまま。

### 起動時の復元フロー

1. `tabsStore.load()` で state 取得(migration 経由)
2. `tabsController` 初期化時、全 pane の全 tab に対して pty spawn
3. renderer 起動時 `tabsGetInitial()` で state 受信
4. 分割状態 / ratio / 各 pane の tabs / active 状態をすべて再現

## 6. ショートカット & 操作

### 新規

| ショートカット | 動作 |
|---|---|
| `Cmd+D` | 分割トグル |

### 既存ショートカットの挙動変更

| ショートカット | 動作 |
|---|---|
| `Cmd+T`(新規タブ) | **フォーカス中 pane** に追加 |
| `Cmd+W`(タブ閉じる) | フォーカス中 pane の active tab を閉じる |
| `Cmd+R`(YouTube reload) | 変更なし(pane と無関係) |
| `Cmd+1/2/3`(モード切替) | 変更なし |

### マウス操作

- pane 内 / tab bar クリック → その pane にフォーカス
- splitter ドラッグ → 比率変更
- タブを他 pane の tab bar にドラッグ → クロスペイン移動

### DnD 詳細

既存の tab DnD(`tabBar.ts` の dragstart/dragover/drop)を拡張:
- dragover で drop 対象のタブがどの pane に属するか検出
- drop 時、同 pane なら `tabsMove`、他 pane なら `tabsMoveAcross` に分岐
- 空 pane への drop(右 pane 末尾など tab が無い領域):tab bar の空白領域を drop zone として扱い、`beforeId = null` で末尾追加

### 視覚フィードバック

- ドラッグ中: 対象 tab が opacity 0.3(既存)
- drop 位置: 2px cyan ライン(既存)
- クロスペイン DnD 中、対象 pane の tab bar に薄い cyan outline(区別用)

## 7. エッジケース

| ケース | 挙動 |
|---|---|
| 分割中に右 pane の最後のタブを閉じる | 自動 unsplit、全 tab が左 pane に |
| 分割中に左 pane の最後のタブを閉じる | 自動 unsplit、右 pane の tabs が唯一の pane に |
| 全体で最後のタブを閉じる | window close(既存動作) |
| タブ DnD で元 pane が空になる | 自動 unsplit |
| 分割なし時 `Cmd+D` | 分割 ON、右に新 tab、右 pane にフォーカス移動 |
| 分割中 `Cmd+D` | unsplit、右 tabs が左末尾に concat、activeId は元フォーカス pane のもの |
| 分割中 `Cmd+T` | フォーカス中 pane に新 tab、その tab が active に |
| 分割中 `Cmd+W` | フォーカス中 pane の active tab を閉じる(上記ルール適用) |
| rename 中のタブを DnD | 既存通り drag 無効 |
| クロスペイン DnD 完了 | 移動先 pane の active に、pane フォーカスも移動先に |
| 分割中にモード切替 | 分割状態は維持、表示だけが mode に従う |

## 8. テスト戦略

### Reducer 単体テスト(`tests/main/tabs.test.ts`)

追加するケース(既存の 25 ケースに加えて):

- `split-panes`: 分割、no-op(既に分割中)、activePaneIndex 変化
- `unsplit-panes`: 統合、no-op(分割なし)、activeId の引き継ぎ
- `activate-pane`: 切替、存在しない index no-op
- `move-tab-to-pane`: 先頭 / 末尾 / 指定 before 挿入、ソース空になる → auto-unsplit
- `close-tab`: pane が空 → auto-unsplit、全体 0 個 → state そのまま
- `new-tab` 分割中: フォーカス pane に追加
- `activate-tab` 他 pane: `activePaneIndex` が切り替わる
- `set-split-ratio`: clamp、分割なし no-op

### Migration テスト(`tests/main/tabsStore.test.ts`)

- 旧 shape JSON(panes なし)→ 新 shape に変換される
- 新 shape JSON → そのまま読める
- 壊れた JSON → INITIAL_TABS_STATE にフォールバック
- ratio が範囲外 → clamp or reset

### Controller テスト(`tests/main/tabsController.test.ts`)

- `toggleSplit()` → id 採番 → action dispatch の流れ
- `moveTabToPane()` API
- pty のライフサイクル: pane 移動で pty は kill しない

### E2E(optional、playwright)

- Cmd+D 実行 → 画面分割、新 tab 生成
- タブを他 pane にドラッグ → 移動
- 再起動後に分割状態が復元される
