# youterm v0.2 設計書 — パーソナライゼーション

**作成日**: 2026-04-18
**対象**: youterm v0.2
**前提**: v0.1.0 がリリース済み

## 1. 概要

v0.2 は v0.1 に「設定保存」「オプションパネル」「ランタイム透過度/背景色変更」を追加し、起動ごとの見た目がユーザー好みで復元されるパーソナライゼーション機能を提供する。

v0.2 のスコープから **広告ブロック**、**プリセットテーマ**、**blur / フォント変更** は除外(それぞれ v0.3 以降)。

## 2. v0.2 スコープ

### 含むもの

- 設定の永続化(`electron-store`): transparency / bgColor / lastMode の 3 項目
- オプションパネル(Cmd+, でトグル、フローティングポップオーバー)
  - 透過度スライダー(0.0〜0.9、step 0.05)
  - 背景色スウォッチ(黒 / ダークグレー / ダークブルー / ダークグリーン の 4 色)
  - Reset ボタン(INITIAL_SETTINGS へ戻す)
- ランタイム反映(スライダー操作中にリアルタイムで背景が変化)
- ショートカット追加: Cmd+] / Cmd+[(透過度増減)、Cmd+,(パネルトグル)

### 含まないもの(v0.3 以降)

- プリセットテーマ(Focus / Chill / Video)
- 広告ブロック
- blur、フォント変更、文字色変更
- 最後に開いていた YouTube URL の復元
- ウィンドウサイズ・位置の復元

### 3.2 レイヤリング方針

v0.1 と同じ 2 層(YouTube + Terminal の WebContentsView)を維持。オプションパネルは Terminal renderer 内の HTML オーバーレイとして実装し、Terminal view と同じ z-index 階層に属する。

**モード依存**:
- パネルは Terminal view 内に居るため、YouTube-only モード中は Terminal view 自体が不可視なのでパネルも見えない
- `Cmd+,` 押下時に main で mode を確認し、YouTube-only なら先に overlay へ遷移 → その後 Terminal renderer にパネルトグル要求を送る

## 4. 型定義

```ts
// src/shared/types.ts 追加分

export type ColorKey = 'black' | 'dark-gray' | 'dark-blue' | 'dark-green'

export interface Settings {
  transparency: number  // 0.0 〜 0.9
  bgColor: ColorKey
  lastMode: Mode
}

export const INITIAL_SETTINGS: Settings = {
  transparency: 0.75,
  bgColor: 'black',
  lastMode: 'overlay',
}

export const COLOR_VALUES: Record<ColorKey, { r: number; g: number; b: number }> = {
  'black':      { r: 0,  g: 0,  b: 0  },
  'dark-gray':  { r: 24, g: 24, b: 24 },
  'dark-blue':  { r: 10, g: 15, b: 35 },
  'dark-green': { r: 10, g: 24, b: 14 },
}
```

## 5. State 管理

### 5.1 純粋 reducer(`src/main/settings.ts`)

```ts
export type SettingsAction =
  | { type: 'set-transparency'; value: number }
  | { type: 'set-color'; color: ColorKey }
  | { type: 'set-last-mode'; mode: Mode }
  | { type: 'reset' }

export function transitionSettings(state: Settings, action: SettingsAction): Settings
```

**ルール**:
- `set-transparency`: clamp 0.0〜0.9。範囲外は clamp、同じ値なら reference equality で state を返す(no-op)
- `set-color`: `COLOR_VALUES` に存在しないキーは state 変わらず
- `set-last-mode`: 同じなら no-op
- `reset`: `INITIAL_SETTINGS` を返す

### 5.2 SettingsController(`src/main/settingsController.ts`)

```ts
export interface SettingsStore {
  load(): Settings
  save(s: Settings): void
}

export interface SettingsController {
  getSettings(): Settings
  dispatch(action: SettingsAction): void
  subscribe(cb: (s: Settings) => void): () => void
}

export function createSettingsController(deps: {
  store: SettingsStore
  debounceMs?: number  // default 200
}): SettingsController
```

- 初期 state は `deps.store.load()`(失敗時は内部で INITIAL_SETTINGS にフォールバック)
- `dispatch` → 新 state に遷移(reference equality で変化判定) → subscribers 通知 → `store.save` を debounce タイマーで遅延実行
- `subscribe` はコールバックを登録し、unsubscribe 関数を返す(初回呼び出しはしない)

### 5.3 electron-store ラッパー(`src/main/settingsStore.ts`)

```ts
export function createRealSettingsStore(): SettingsStore
```

- `new ElectronStore({ name: 'settings', defaults: INITIAL_SETTINGS })` を内部で生成
- `load()`: `store.store` を返す。schema 検証(transparency が number で 0-0.9、bgColor が有効 key、lastMode が有効 Mode)に失敗したら INITIAL_SETTINGS
- `save(s)`: `store.set(s)` で全件上書き

保存先: electron-store デフォルトの `app.getPath('userData')/settings.json`(mac では `~/Library/Application Support/youterm/settings.json`)

## 6. IPC 仕様

### main → renderer

| チャネル | payload | 送信先 | タイミング |
|---|---|---|---|
| `settings:changed` | `Settings` | Terminal renderer | settings 変更時(debounce 前、即時) |
| `panel:toggle` | なし | Terminal renderer | `Cmd+,` 押下時(main が overlay 遷移後に送信) |

### renderer → main

| チャネル | payload | 送信元 | 効果 |
|---|---|---|---|
| `settings:set-transparency` | `number` | Terminal | reducer dispatch |
| `settings:set-color` | `ColorKey` | Terminal | reducer dispatch |
| `settings:reset` | なし | Terminal | reducer dispatch |

### invoke(双方向)

| チャネル | payload | 送信元 | 返り値 |
|---|---|---|---|
| `settings:get-initial` | なし | Terminal | `Settings`(起動時の初期値、CSS 適用前に同期取得) |

### preload API 拡張

```ts
interface YoutermAPI {
  // 既存(v0.1)
  onStateChanged(cb: (s: AppState) => void): () => void
  onPtyData(cb: (d: string) => void): () => void
  ptyWrite(d: string): void
  ptyResize(size: { cols: number; rows: number }): void

  // v0.2 追加
  onSettingsChanged(cb: (s: Settings) => void): () => void
  onPanelToggle(cb: () => void): () => void
  settingsGetInitial(): Promise<Settings>
  settingsSetTransparency(value: number): void
  settingsSetColor(color: ColorKey): void
  settingsReset(): void
}
```

## 8. Options Panel UI

### 8.1 配置

- `position: fixed; top: 48px; right: 12px;`(モードインジケータの下)
- 幅 280px、高さは内容に応じて自動(〜260px)
- z-index 100(xterm より前)

### 8.2 DOM 構造

```html
<div id="options-panel" class="is-closed">
  <div class="options-section">
    <label>Transparency</label>
    <input type="range" id="opt-transparency" min="0" max="0.9" step="0.05" />
    <span class="opt-value">0.75</span>
  </div>

  <div class="options-section">

      <button class="swatch" data-color="dark-gray"  aria-label="Dark gray"></button>
      <button class="swatch" data-color="dark-blue"  aria-label="Dark blue"></button>
      <button class="swatch" data-color="dark-green" aria-label="Dark green"></button>
    </div>
  </div>

  </div>

```

### 8.3 挙動

- **開閉**: `panel:toggle` IPC → `.is-open` / `.is-closed` クラストグル
- **Esc**: パネル開いている時のみ閉じる(document keydown リスナー)
- **パネル外クリック**: 閉じる(document click、パネル内は stopPropagation)
- **フォーカス**: 開いた時に transparency slider を focus、閉じた時は xterm に戻す
- **スライダー操作**: `input` イベント(ドラッグ中も発火)→ `settingsSetTransparency`
- **swatch クリック**: `settingsSetColor` → `.is-active` クラスは `onSettingsChanged` 受信時に更新(main が truth source)

### 8.4 背景への反映

`#terminal-root` の background を CSS 変数ベースに変更:

```css
:root {
  --term-bg-r: 0;
  --term-bg-g: 0;
  --term-bg-b: 0;
  --term-alpha: 0.75;
}

#terminal-root {
  background: rgba(var(--term-bg-r), var(--term-bg-g), var(--term-bg-b), var(--term-alpha));
}
```

`onSettingsChanged` リスナーで:

```ts
const { r, g, b } = COLOR_VALUES[settings.bgColor]
document.documentElement.style.setProperty('--term-bg-r', String(r))
document.documentElement.style.setProperty('--term-bg-g', String(g))
document.documentElement.style.setProperty('--term-bg-b', String(b))
document.documentElement.style.setProperty('--term-alpha', String(settings.transparency))
```

起動時のフリッカー防止:
- renderer mount 直後に `settingsGetInitial()` を await
- CSS 変数を適用してから xterm を mount、パネル初期化

## 9. 起動シーケンス

```
app.whenReady()
  1. bundle = createMainWindow()
  2. await attachPty(bundle.terminalView)
  3. settings = createSettingsController({ store: createRealSettingsStore() })
  4. modeCtrl = createModeController(bundle, { initialMode: settings.getSettings().lastMode })
  5. modeCtrl.subscribe(state => {
       if (state.mode !== settings.getSettings().lastMode) {
         settings.dispatch({ type: 'set-last-mode', mode: state.mode })
       }
     })
  6. settings.subscribe(s => bundle.terminalView.webContents.send('settings:changed', s))
  7. installShortcuts(bundle, modeCtrl, settings)
  8. IPC ハンドラ登録(settings:set-transparency / set-color / reset)
  9. ipcMain.handle('settings:get-initial', () => settings.getSettings())
```

## 10. エラーハンドリング

| ケース | 挙動 |
|---|---|
| `settings.json` 読み込み失敗 | INITIAL_SETTINGS にフォールバック、`console.error` |
| `settings.json` schema 不整合 | 同上 |
| `settings.json` 書き込み失敗 | `console.error` のみ、in-memory 値は保持 |
| IPC で不正な型/値 | reducer が弾く(state 変わらず)、書き込まれないので副作用なし |
| CSS 変数書き換え失敗 | try/catch で無視 |
| debounce 中にアプリクラッシュ | 保存されない変更があり得る(200ms 以内の変更)、許容 |

## 11. テスト戦略

### ユニット(vitest、TDD)

**`src/main/settings.ts`**(reducer):
- 全 action 網羅
- clamp 挙動(-0.1 → 0, 1.0 → 0.9, 0.5 → 0.5)
- 不正 ColorKey / Mode の扱い
- reference equality idempotency
- reset 挙動

**`src/main/settingsController.ts`**:
- initial state が store.load から来る
- dispatch で state 遷移 + subscribers 通知
- store.save が debounce される(fake timers)
- 複数 dispatch 中は最後の state が 1 回だけ保存される

**`src/main/settingsStore.ts`**:
- schema validation(transparency 範囲、bgColor 妥当性、lastMode 妥当性)
- 不正データ時の INITIAL_SETTINGS フォールバック

予想テスト数: 18 前後

### E2E

v0.1 のスモーク E2E(`tests/e2e/smoke.test.ts`)をそのまま維持。設定保存の E2E は書かない(手動テストで足りる)。

### 手動テストチェックリスト

- `Cmd+,` でパネル開閉
- Esc でパネル閉じる
- パネル外クリックで閉じる
- スライダー操作中に背景透過度が即座に追随
- 色 swatch クリックで背景色が変わる、active 枠移動
- `Cmd+]` / `Cmd+[` で透過度が 0.05 刻みで変化、パネルの数値表示も同期
- `Reset` で INITIAL_SETTINGS 状態に戻る
- youtube-only モード中に `Cmd+,` → overlay モードに遷移してパネル表示
- アプリ再起動で transparency / bgColor / lastMode が復元される
- `~/Library/Application Support/youterm/settings.json` を手動削除 → 起動時 INITIAL_SETTINGS にフォールバック
