# youterm v0.1 設計書

**作成日**: 2026-04-17
**対象**: youterm v0.1 (MVP)
**プラットフォーム**: macOS 専用

## 1. 概要

youterm は YouTube を背景動画レイヤーとして常時再生しつつ、その上に半透明のターミナル UI を重ねて利用できる macOS デスクトップアプリ。Claude Code を起動しながらお気に入りの YouTube MV / プレイリストを流す用途を想定する。

v0.1 は「気持ち良く動く最小構成」を目的とし、視覚的な体験とモード切替の核だけを実装する。設定保存・広告ブロック・オプションパネルは v0.2 以降に先送り。

## 2. v0.1 スコープ

### 含むもの

- Electron ウィンドウ生成(標準 mac 枠・1280×800・リサイズ可)
- 背景に YouTube を WebContentsView で表示(`https://www.youtube.com/` 固定)
- 前面に半透明のターミナルオーバーレイ(xterm.js + node-pty で zsh 接続)
- 3モード切替: YouTube 専用 / 重ね表示 / ターミナル専用
- 入力先切替: terminal ↔ youtube(重ね表示モード時のみ有効)
- キーボードショートカットによる操作
- モード・入力先の状態は main プロセスが単一の source of truth

### 含まないもの(v0.2 以降)

- 設定保存 / 復元
- 広告ブロック(session は分離して実装するが、adblock ライブラリは未適用)
- オプションパネル(透過度スライダー / 背景色プリセット / プリセットテーマ)
- 透過度・背景色の動的変更(v0.1 では定数)
- blur、フォント変更、テーマエディタ
- 複数ターミナル、タブ、分割ペイン
- 起動時 URL の永続化
- Windows / Linux 対応
- `.app` パッケージング(v0.1 は開発ビルドのみ)

## 3. 技術選定

| 項目 | 選定 | 備考 |

## 4. アーキテクチャ

### 4.1 プロセス構成

```
┌─ Main process (Node.js) ─────────────────────────────┐
│  - BrowserWindow 生成 (標準 mac 枠・1280×800)        │
│  - 2つの WebContentsView を addChildView             │
│  - node-pty で zsh を spawn                          │
│  - Application Menu + accelerator でショートカット   │
│  - AppState を保持 (source of truth)                 │
│  - IPC ルーティング                                   │
└───────────────────────────────────────────────────────┘
         │                                    │
         ▼                                    ▼
┌─ YouTube WebContentsView ────┐   ┌─ Terminal WebContentsView ──┐
│  session: persist:youtube    │   │  session: persist:terminal    │
│  Preload: youtube.ts (空)    │   │  Preload: terminal.ts         │
│  URL: https://www.youtube.com│   │  Renderer: xterm.js + UI      │
│  sandbox: true               │   │  背景: rgba(0,0,0,0.75)       │
└──────────────────────────────┘   └───────────────────────────────┘
```

### 4.2 レイヤリング方式

- BrowserWindow のルート `contentView` に 2 つの WebContentsView を `addChildView` で追加
- 追加順: YouTube → Terminal(Electron は追加順に上へ重なる)
- BrowserWindow 本体の webContents は使用しない(空の `index.html` をロード)
- 両 view の session を `persist:youtube` / `persist:terminal` で分離(将来の adblock 適用に備える)

### 4.3 モード → view マッピング

| mode | YouTube view bounds | Terminal view bounds | inputTarget デフォルト | focus 先 |
|---|---|---|---|---|
| `youtube-only` | フル | `{0,0,0,0}` | `youtube` | YouTube webContents |
| `overlay` | フル | フル(背景透過) | `terminal` | Terminal webContents |
| `terminal-only` | `{0,0,0,0}` | フル | `terminal` | Terminal webContents |

- `{0,0,0,0}` は view を不可視にするが、webContents は生きているため YouTube の再生は継続する
- focus は `webContents.focus()` で明示的に切り替える

## 5. 状態管理

```ts
type Mode = 'youtube-only' | 'overlay' | 'terminal-only'
type InputTarget = 'youtube' | 'terminal'

interface AppState {
  mode: Mode
  inputTarget: InputTarget
}

const INITIAL_STATE: AppState = {
  mode: 'overlay',
  inputTarget: 'terminal',
}
```

- main プロセスが単一の状態保持者
- 状態変更は main 内のトランジション関数を経由(テスト可能な純粋関数)
- 変更後は両 renderer へ `state:changed` をブロードキャスト
- **モード遷移時の inputTarget**: 遷移先モードの「デフォルト inputTarget」(§4.3 表)に自動的に設定される。overlay から離れる時は inputTarget がモード固有の値に上書きされる
- **入力先切替(`Cmd+\`)**: `overlay` モード時のみ effective。他モードでは no-op(1つしか可視 view がないため切替の意味がない)

## 6. IPC 仕様

### main → renderer(broadcast)

| チャネル | payload | 送信先 | タイミング |
|---|---|---|---|
| `state:changed` | `{ mode, inputTarget }` | 両 renderer | 状態変更時 |
| `pty:data` | `string` (UTF-8) | Terminal renderer | pty stdout/stderr |

### renderer → main

| チャネル | payload | 送信元 | 効果 |
|---|---|---|---|
| `pty:write` | `string` | Terminal | pty stdin 書き込み |
| `pty:resize` | `{ cols, rows }` | Terminal | pty リサイズ |

v0.1 ではモード切替・入力先切替のトリガは全て Application Menu の accelerator なので main プロセス内で完結する。renderer から状態変更を起こす IPC は持たない(v0.2 でオプションパネルを追加する際に追加する)。

### Terminal preload が露出する API (contextBridge)

```ts
window.youtermAPI = {
  onStateChanged(handler: (state: AppState) => void): void
  onPtyData(handler: (data: string) => void): void
  ptyWrite(data: string): void
  ptyResize(size: { cols: number; rows: number }): void
}
```

### YouTube preload

- 空(contextBridge も状態受信もしない)
- `contextIsolation: true`, `nodeIntegration: false`, `sandbox: true`

## 7. ショートカット

Application Menu の accelerator として登録(アプリがフォーカスされている時のみ有効)。

| Accelerator | 動作 |
|---|---|
| `Cmd+1` | mode → `youtube-only` |
| `Cmd+2` | mode → `overlay` |
| `Cmd+3` | mode → `terminal-only` |
| `Cmd+\` | inputTarget トグル(overlay 時のみ effective) |
| `Cmd+R` | YouTube view 再読み込み |
| `Cmd+Shift+R` | アプリ全体ハードリロード |

v0.2 以降: `Cmd+]`, `Cmd+[`, `Cmd+,`(透過度・オプションパネル関連)。

## 8. シェル / pty

- `node-pty.spawn('/bin/zsh', ['-l'], { cwd: os.homedir(), env })`
  - `-l` = ログインシェル(`.zprofile` 経由で PATH 等を読み込む)
  - `env` は `process.env` をベースに `TERM=xterm-256color`, `COLORTERM=truecolor` を上書き
- pty は 1 本のみ(複数ペイン・タブは v0.2 以降)
- pty の `onData` を `pty:data` で renderer へ
- renderer の xterm `onData` を `pty:write` で main へ
- ウィンドウリサイズ → xterm FitAddon → `pty:resize` → pty.resize(cols, rows)

### xterm.js 設定

```ts
new Terminal({
  allowTransparency: true,
  theme: { background: 'rgba(0,0,0,0)' },
  fontFamily: 'Menlo, monospace',
  fontSize: 13,
  cursorBlink: true,
  scrollback: 10000,
})
```

- FitAddon, WebLinksAddon を load
- 背景色は xterm 自体には持たせず、外側の HTML コンテナに `background: rgba(0,0,0,0.75)` を指定

## 11. テスト戦略

### ユニット (vitest)

- **`src/main/state.ts`** — mode 遷移・inputTarget トグルの純粋ロジックを全遷移網羅
- **`src/main/pty.ts`** — spawn 関数を依存注入で差し替え可能にし、zsh 無しでインターフェース層をテスト

### スモーク E2E (playwright-electron)

- 1 シナリオのみ: アプリ起動 → `echo hello\n` を terminal に送信 → 出力に `hello` が含まれる

### 手動テスト(設計書のチェックリストで残す)

- モード遷移で YouTube の再生が途切れないこと
- `rgba(0,0,0,0.75)` で動画が透けつつ文字が読めること
- Cmd+1/2/3 の体感速度
- ウィンドウリサイズで xterm と pty のサイズが追従すること
- Claude Code が overlay モード内の terminal で正常に動作すること

### 方針

- TDD は `state.ts` にのみ厳格適用
- UI / 描画層の自動テストは書かない(中途半端なテストは保守コストだけ残る)
- E2E は壊れやすいので最小 1 本に限定

## 12. セキュリティ

- 両 WebContentsView で `contextIsolation: true`, `nodeIntegration: false`
- YouTube view は `sandbox: true`(外部サイトをロードするため)
- Terminal view の preload は contextBridge 経由のみ API を露出、`ipcRenderer` を直接触らせない

## 13. v0.2 以降の拡張方針(参考)

仕様書原典(ChatGPT 作成)の以下を v0.2 以降で段階的に取り込む:

1. **設定保存**(electron-store) — 最後のモード、起動時 URL、ウィンドウ位置

3. **広告ブロック**(YouTube session への adblock 適用、ON/OFF 切替)
4. **プリセットテーマ**(Focus / Chill / Video)
5. **blur、フォント変更**
6. **Cmd+], Cmd+[, Cmd+,** ショートカット追加
