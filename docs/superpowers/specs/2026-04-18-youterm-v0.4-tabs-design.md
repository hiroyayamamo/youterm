# youterm v0.4 設計書 — Terminal Tabs

**作成日**: 2026-04-18
**対象**: youterm v0.4.0
**前提**: v0.3.1 がリリース済み

## 1. 概要

ターミナルレイヤーに複数タブを持てるようにする。各タブは独立した zsh pty + xterm インスタンス。ブラウザ標準的な UX(Cmd+T で新規、×ボタンで閉じる、右クリックで menu、リネーム可能)。背景の YouTube iframe はタブ機能の対象外・変更なし。

## 2. スコープ

### 含む

- タブバー(`#terminal-root` 上部、高さ 28px)
- `Cmd+T`: 新規タブ作成
- `Cmd+W`: アクティブタブを閉じる(タブ 1 個の時のみウィンドウを閉じる)
- `Ctrl+Tab` / `Ctrl+Shift+Tab`: 次 / 前のタブへ循環切替
- タブクリックで切替
- タブの × ボタンでクローズ
- タブ上で右クリック → コンテキストメニュー(「Close tab」「Rename tab」)
- タブ名のリネーム(インプレース入力、Enter 確定 / Esc キャンセル / blur 確定)
- 閉じる前の確認(zsh に子プロセスがある場合のみ、ネイティブダイアログで)
- タブ幅の制約: `min-width: 80px`, `max-width: 200px`, 文字は末尾省略(`text-overflow: ellipsis`)
- 大量タブ時の横スクロール(スクロールバー非表示)

### 含まない(v0.5 以降)

- タブの永続化(起動時に前回のタブ構成を復元)
- タブの並び替え(ドラッグ)
- タブごとの cwd 追従
- タブごとの個別背景色や透過度
- 分割ペイン

## 3. データモデル

### `src/shared/types.ts` に追加

```ts
export interface Tab {
  id: string           // UUID or monotonic string ID
  customName: string | null  // null なら "zsh" がデフォルト表示
}

export interface TabsState {
  tabs: Tab[]          // 順序(表示順)
  activeId: string     // アクティブタブの ID
}

export const INITIAL_TABS_STATE: TabsState = {
  tabs: [{ id: '1', customName: null }],
  activeId: '1',
}
```

タブ ID はシンプルにモノトニックな文字列(`"1"`, `"2"`, ...)。起動時は 1 タブ。

## 4. アーキテクチャ

### 4.1 タブ state の場所

Main プロセスが **source of truth**。理由:
- pty プロセスの lifecycle 管理が main なので、タブの生成/破棄と同期する必要がある
- ネイティブダイアログ(閉じ確認)は main プロセスから
- renderer は UI を描画するだけ

### 4.2 Per-tab pty

- `src/main/tabsController.ts`(新規): タブ list 管理 + pty map
- タブ作成時に `createPtyHandle(...)` を呼んで pty spawn
- タブ破棄時に `ptyHandle.kill()` で pty 停止
- `tabId → PtyHandle` の Map を保持

### 4.3 IPC

既存の pty:* チャネルに tabId を入れる形にリファクタ。

**main → renderer**:
| チャネル | payload |
|---|---|
| `tabs:state` | `TabsState`(初期 & 変更時ブロードキャスト) |
| `pty:data` | `{ tabId: string, data: string }` |

**renderer → main**:
| チャネル | payload | 目的 |
|---|---|---|
| `pty:write` | `{ tabId: string, data: string }` | pty stdin 書き込み |
| `pty:resize` | `{ tabId: string, cols: number, rows: number }` | pty リサイズ |
| `tabs:new` | なし | 新規タブ作成要求 |
| `tabs:close` | `{ tabId: string }` | クローズ要求(main が process チェック → ダイアログ → 実行) |
| `tabs:activate` | `{ tabId: string }` | アクティブ切替 |
| `tabs:rename` | `{ tabId: string, name: string \| null }` | カスタム名設定(null でクリア) |
| `tabs:context-menu` | `{ tabId: string, x: number, y: number }` | 右クリック位置でメニュー展開 |

**invoke**:
| チャネル | payload | 戻り値 |
|---|---|---|
| `tabs:get-initial` | なし | `TabsState` |

### 4.4 Preload API 追加

```ts
interface YoutermAPI {
  // 既存...

  // tabs
  onTabsState(cb: (state: TabsState) => void): () => void
  tabsGetInitial(): Promise<TabsState>
  tabsNew(): void
  tabsClose(tabId: string): void
  tabsActivate(tabId: string): void
  tabsRename(tabId: string, name: string | null): void
  tabsContextMenu(tabId: string, x: number, y: number): void

  // pty(既存から変更: tabId 付き)
  onPtyData(cb: (data: { tabId: string; data: string }) => void): () => void
  ptyWrite(tabId: string, data: string): void
  ptyResize(tabId: string, size: { cols: number; rows: number }): void
}
```

**既存の onPtyData / ptyWrite / ptyResize の payload が変わる破壊的変更**。renderer 側を全面更新する必要あり。

## 5. 閉じる前の確認

### 5.1 「動いているプロセス」判定

`tabs:close` 受信時、main で:

```ts
import { exec } from 'node:child_process'

function hasChildProcesses(zshPid: number): Promise<boolean> {
  return new Promise(resolve => {
    exec(`pgrep -P ${zshPid}`, (err, stdout) => {
      if (err && err.code !== 1) {
        // pgrep error other than "no match" — assume no children to avoid blocking
        resolve(false)
        return
      }
      resolve(stdout.trim().length > 0)
    })
  })
}
```

`pgrep -P <pid>` は指定 pid を親とする子プロセスを列挙。exit code 1 は「マッチなし」を意味する(エラーではない)。

### 5.2 ダイアログ

子プロセスがある場合、`dialog.showMessageBox`:

```ts
const { response } = await dialog.showMessageBox(win, {
  type: 'warning',
  message: 'Close this tab?',
  detail: 'A process is still running. Closing will terminate it.',
  buttons: ['Close', 'Cancel'],
  defaultId: 1,
  cancelId: 1,
})
if (response === 0) proceedToClose()
```

## 6. UI

### 6.1 タブバー

```html
<div id="tab-bar">
  <div id="tab-list">
    <div class="tab" data-tab-id="1">
      <span class="tab-label">zsh</span>
      <button class="tab-close" aria-label="Close tab">×</button>
    </div>
    <!-- more tabs -->
    <button id="tab-new" aria-label="New tab">+</button>
  </div>
</div>
```

- `#tab-bar`: `position: absolute; top: 0; left: 0; right: 0; height: 28px; display: flex;`
- `#tab-list`: `flex: 1; display: flex; overflow-x: auto;`(overflow でスクロール)
- `.tab`: `min-width: 80px; max-width: 200px; display: flex; align-items: center; padding: 0 8px; border-right: 1px solid rgba(255,255,255,0.1); cursor: pointer;`
- `.tab.is-active`: 背景が薄く明るい、下線
- `.tab-label`: `flex: 1; text-overflow: ellipsis; overflow: hidden; white-space: nowrap;`
- `.tab-close`: 右側の×ボタン、`.tab:hover` で表示(非アクティブタブは常時は非表示)
- `#tab-new`: `+` ボタン、タブリストの末尾

既存 `#terminal-inner` の位置調整:
```css
#terminal-inner {
  top: 40px;  /* was 12px → タブバー(28px)+ 12px margin */
  left: 12px;
  right: 12px;
  bottom: 36px;
}
```

### 6.2 コンテキストメニュー

タブ上の右クリックで renderer が `x, y` を送信 → main が `Menu.popup()` で ネイティブメニュー表示。メニュー項目:
- "Close tab" → `tabs:close` 発火(pgrep チェック通る)
- "Rename tab" → renderer に `tabs:start-rename` IPC、renderer 側でインプレース入力開始

### 6.3 リネーム入力

タブラベルの `<span>` を `<input>` に差し替え:
- 幅: タブ領域いっぱい(label と同じ)
- autofocus、初期値 = 現在のカスタム名 or ""(プレースホルダに "zsh")
- Enter: 確定、`tabsRename` IPC 送信、span に戻す
- Esc: キャンセル、span に戻す
- blur: 確定(Enter と同じ扱い)
- 空文字で確定 → customName = null(デフォルトに戻る)

## 7. ショートカット

Application Menu の "Edit" or 新規 "Tab" サブメニューに追加。

| Accelerator | 動作 |
|---|---|
| `Cmd+T` | 新規タブ(`tabsController.newTab()`) |
| `Cmd+W` | アクティブタブ閉じる。タブ 1 個なら `window.close()` |
| `Ctrl+Tab` | 次タブ(循環) |
| `Ctrl+Shift+Tab` | 前タブ(循環) |
| `Cmd+Shift+]` | 次タブ(Chrome 風、二次受付) |
| `Cmd+Shift+[` | 前タブ |

既存の Cmd+[ / Cmd+] は透過度操作なので **Shift 付き**で区別する。

## 8. モード統合

- `youtube-only`: Terminal overlay 全体が `display: none` → タブバーも消える(現状の挙動のまま)
- `overlay` / `terminal-only`: タブバー見える
- モード切替中も裏でタブの pty は全て稼働し続ける(出力ロストしない)

## 9. modeIndicator の位置

現在タブバー位置にあるモードインジケータ(右上 `top: 8px, right: 12px`)と衝突する。移設:
- **提案**: タブバー右端(`#tab-bar` 内の右寄せ。タブリストより右に置く)
- or: Footer 領域(タブ下部)に移す → 既存の `bottom: 36px` に収める
- **採用**: タブバー右端に移設。DOM 的には `#tab-bar > #mode-indicator`(flex で右寄せ)

## 10. テスト戦略

### ユニット(vitest、TDD)

- **`src/main/tabs.ts`**(新規 reducer、純粋関数):
  - `TabsAction`: `new-tab` / `close-tab` / `activate-tab` / `rename-tab`
  - `transitionTabs(state, action)` の全遷移網羅
  - ID 割当の単調性、クローズ時のアクティブ切替(次が無ければ前)
  - 最後のタブをクローズ要求時: state は空にせず現状維持(main がウィンドウ閉じる方の分岐)
  - 予想テスト数: 12 前後

- **`src/main/tabsController.ts`**:
  - DI で pty ファクトリ注入、タブ作成/破棄で spawn/kill が呼ばれる
  - close-with-check が `hasChildProcesses` を問う → 結果に応じてダイアログ or 即クローズ
  - `hasChildProcesses` 関数は DI(本番は pgrep、テストは fake)
  - 予想テスト数: 8 前後

### E2E / 手動

- スモーク E2E(既存)は維持
- 手動: タブ作成、切替、閉じる、リネーム、子プロセスありでの確認ダイアログ、モード切替でのタブバー表示/非表示

## 11. 破壊的変更

- `onPtyData` / `ptyWrite` / `ptyResize` の payload 変更(tabId 付きになる)
- `attachPty` は `attachTabs` に置き換えられ、内部で tab 管理 + pty map を持つ

旧 API は残さず、一気に置き換え。renderer main.ts も破壊的変更で対応。
